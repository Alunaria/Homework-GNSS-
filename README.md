## Основная часть 1
В идеальном случае местоположение приёмника - точка пересечения всех окружностей (где центр окружности - координаты маяка, радиус - расстояние от маяка до приёмника). Однако из-за неточности часов и помех полученные расстояния от маяков до приёмника не являются истинными и называются псевдорасстояниями - расстояниями с погрешностью ввиду ошибки точности измерения времени.

### Алгоритм поиска координат

1. Для всех псевдодальностных окружностей находятся точки их пересечения путём решения систем из двух уравнений. При этому? если расстояние между центрами окружностей больше суммы их радиусов, то пересечения не наблюдается. Затем находятся средние значения по координатам x и y, которые и будут являться предполагаемой позицией (красная точка на рисунке ниже).
![pic_1](https://github.com/Alunaria/Homework-GNSS-/raw/main/image/pic_1.png)
Из рисунка заметно, что одна точка пересечения, расположенная довольно далеко от скопления остальных, существенно влияет на вычисленные координаты.

2. Составление матричного уравнения
Обозначения:
psri - псевдорасстояния
ri - расстояния до предполагаемой позиции
   - погрешность часов, переведенная к единицам длины
dх - погрешность по координате х
dу - погрешность по координате у
х0, у0  - координаты предполагаемой позиции объекта (среднее по координатам)

Тогда настоящие координаты приёмника вычисляются следующим образом:

- x = x0 + dx
- y = y0 + dy

Уравнение окружности нелинейно, применяется разложение в ряд Тейлора, берутся первые два слагаемых:

![formula_4](https://github.com/Alunaria/Homework-GNSS-/raw/main/image/formula_4.png)

![formula_5](https://github.com/Alunaria/Homework-GNSS-/raw/main/image/formula_5.png)

Mатрица A^(-1)  является псевдообратной. Вычисляется матрица С, содержащая в себе ошибки по координатам и времени. Пересчитывается предполагаемая позиция. Последние 2 операции проделываются n раз для минимизации невязок. После совершения m итерация (при m<n) минимизация достигает своего предела при n от ~100.
